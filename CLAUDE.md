# Peekmail - Project Context for Claude Code

## What is Peekmail?
A native macOS menu bar Gmail client built in Swift. Replaces "Made for Gmail" by Fiplab which stopped working (blank WebView). Built entirely from scratch with no external dependencies. Supports multiple Gmail accounts with isolated sessions.

## Architecture

### Core Tech
- **Swift + SwiftUI + AppKit** (no external packages)
- **WKWebView** for Gmail rendering with persistent sessions
- **Gmail Atom feed** (`https://mail.google.com/mail/feed/atom`) for unread count polling every 5 seconds
- **UNUserNotificationCenter** for desktop notifications
- **NSStatusItem** for menu bar presence with custom-drawn envelope icon
- **NSDockTile** badge for unread count on dock icon
- **Multi-account support** with isolated `WKWebsiteDataStore` per account
- `LSUIElement = true` in Info.plist to hide from dock by default
- App Sandbox with network.client entitlement
- Build with: `xcodebuild -scheme Peekmail -configuration Debug -derivedDataPath build`
- Launch with: `open build/Build/Products/Debug/Peekmail.app`

### Key Design Decisions
- **Atom feed over title parsing**: Originally parsed Gmail page title for unread count, but this broke when navigating to Drafts/Sent. Atom feed works regardless of which Gmail view is active.
- **First-poll seeding**: `hasCompletedFirstPoll` flag prevents notification spam on app launch. First poll silently seeds known email IDs; subsequent polls notify for new IDs only.
- **Cookie extraction**: Atom feed auth uses cookies extracted from WKWebView's data store, passed as Cookie header in URLRequest.
- **Isolated data stores**: Each Gmail account gets its own `WKWebsiteDataStore(forIdentifier: UUID)` for cookie isolation. New accounts start with `isNew: true` and load the Gmail sign-in page instead of `mail.google.com`.
- **Manual NSWindow for settings**: SwiftUI `Settings` scene doesn't work with `.accessory` activation policy, so settings uses a manually created NSWindow with NSHostingView.
- **Window hide vs quit**: Cmd+W / close button hides window (app stays in menu bar). Cmd+Q quits.
- **Window follows active Space**: `window.collectionBehavior = [.moveToActiveSpace]` ensures the window opens on the current desktop space, not the space it was last visible on.
- **Icon generation**: `generate_icon.swift` draws directly to CGContext (not NSImage.lockFocus) to avoid retina scaling issues that caused grey borders. DO NOT modify the icon generation approach â€” any changes to the drawing method or dimensions causes a grey border to appear around the app icon. The only PNGs that work without the border are those generated by the committed `generate_icon.swift`.
- **Custom menu bar icon**: SF Symbols `envelope` had uneven stroke weights (thin bottom edge). Replaced with custom-drawn `drawEnvelopeIcon()` in AppDelegate using NSBezierPath for uniform 1.5pt line weight. Filled version (unread state) uses white fill with transparent V flap cutout; `isTemplate = false` for the filled state so macOS doesn't flatten the colors.
- **Google domain allowlist**: GmailWebView's navigation delegate allows Google-owned domains (google.com, googleapis.com, gstatic.com, googleusercontent.com, youtube.com, google-analytics.com, googletagmanager.com, googlesyndication.com, googleadservices.com) to load in-app for main-frame and subframe navigation. Non-Google URLs open in the default browser.
- **Email link clicks always open in default browser**: Clicked links in emails arrive as new-window requests (`targetFrame == nil` â€” Gmail marks them `target="_blank"`) and are ALWAYS sent to the default browser, even Google destinations like Docs/Slides/YouTube. Gmail wraps email-body links in its safe-redirect URL (`google.com/url?q=<dest>`); `unwrapGoogleRedirect()` extracts the real destination before deciding. Exception: `accounts.google.com` / `mail.google.com` popups stay in-app so sign-in flows land in the account's isolated data store.

### Multi-Account System (AccountManager.swift)
- `AccountManager` is an `ObservableObject` with `@Published accounts` array and `selectedIndex`
- Each `GmailAccount` has: UUID, WKWebView, WKWebsiteDataStore, email, profileImageData, unreadCount, isNew flag
- Accounts persist via UserDefaults (account IDs saved, data stores survive across launches)
- Profile images extracted from Google account page via JavaScript injection
- Email address detected from Gmail page title or Google account URL
- New accounts (no email yet) show person silhouette icon; logged-in accounts show profile photo or initial
- Right-click account avatar â†’ context menu with email address and "Remove Account" option
- Removing an account clears its WKWebsiteDataStore

### Account Sidebar (MainWindowView.swift)
- Light blue-grey sidebar (`#EEF1F7`) with 80pt width
- 40pt top spacer for macOS traffic light buttons
- Account avatars: 44px circles with 50px blue selection ring
- Unread badge: red capsule with white count text, offset to top-right
- Dashed circle "+" button to add new accounts
- Bottom: reload and settings gear icons
- Reload/settings buttons use `sidebarIconColor` for consistency

### How Feed Polling Works (AppDelegate.swift)
1. Timer fires every **5 seconds**
2. For each account: skip if no email detected yet (not logged in)
3. Extract cookies from account's WKWebsiteDataStore
4. Fetch `https://mail.google.com/mail/feed/atom` with Cookie header
5. Parse XML response: `<fullcount>` for total unread, `<entry>` elements for individual emails
6. Compare entry IDs against per-account `notifiedEmailIds` set
7. New IDs â†’ send rich notification (sender, subject, snippet) + play alert sound
8. Update menu bar icon (white filled envelope + count, or grey outline for 0)
9. Update dock badge (`NSDockTile.badgeLabel`) with total unread across all accounts

### Menu Bar Icon States
- **No unread**: Grey outline envelope, `isTemplate = true` (adapts to light/dark menu bar)
- **Unread**: White filled envelope with transparent V flap cutout, `isTemplate = false`, black count text next to icon

### Notification System
- Rich notifications show: sender name (title), subject (subtitle), email snippet (body)
- `content.sound = nil` â€” app handles sound separately via NSSound
- `willPresent` returns `[.banner, .list]` (show even when app is in foreground)
- 5 alert sounds: Glass, Ping, Pop, Purr, Tink (configurable in Settings)
- Clicking a notification: switches to the correct account, shows the main window, navigates to the specific email thread via `#inbox/messageId` URL
- Notifications include `accountId` in userInfo for multi-account routing

### Dock Badge
- Shows total unread count across all accounts
- Updated on every poll cycle via `NSApp.dockTile.badgeLabel`
- Must be set on main thread with `NSApp.dockTile.display()` to force refresh
- Only visible when "Show in Dock" is enabled in Settings

### Email CTA Links Open In-App â€” Fixed (2026-06-11, v1.2)
Clicking a CTA/link in an email loaded the page inside Peekmail instead of the default browser. ## Known Issues

### App Icon Grey Border
Any modification to the icon PNGs (different drawing dimensions, different source images, sips-resized images) causes a grey border/frame to appear behind the app icon. Root cause unknown but related to how Xcode's asset catalog compiler processes PNGs. **Do not attempt to update the app icon** â€” the only working PNGs are those generated by the committed `generate_icon.swift` with its exact drawing parameters.
